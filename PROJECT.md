# EvenOddBasketBot — архитектура проекта

## Назначение

EvenOddBasketBot — Telegram-бот для автоматического поиска LIVE-баскетбольных матчей по стратегии нечётного итогового тотала.

Бот:

- получает данные матчей;
- проверяет условия стратегии;
- отправляет сигналы в Telegram;
- сохраняет сигналы в базу данных;
- проверяет итоговый результат;
- определяет WIN или LOSE;
- собирает статистику;
- работает круглосуточно на Render.

## Основные модули

### `main.py`

Главный файл программы.

Отвечает за:

- запуск бота;
- главный цикл;
- проверку новых матчей;
- отправку сигналов;
- проверку завершённых матчей;
- запуск web-сервера для Render.

---

### `parser.py`

Получает данные с Flashscore.

Отвечает за:

- загрузку LIVE-матчей;
- разбор стран и лиг;
- получение счёта;
- получение четвертей;
- определение статуса матча;
- поиск матча по ID.

---

### `strategy.py`

Проверяет условия стратегии.

Сигнал появляется, если:

- первая четверть имеет определённую чётность;
- вторая четверть имеет такую же чётность;
- третья четверть имеет такую же чётность;
- матч перешёл в четвёртую четверть.

---

### `telegram_sender.py`

Отправляет сообщения через Telegram Bot API.

Поддерживает:

- одного или нескольких получателей;
- сигналы;
- результаты;
- уведомления о запуске;
- уведомления об ошибках.

---

### `notifications.py`

Формирует текст сообщений.

Содержит шаблоны:

- сообщения о запуске;
- сообщения об ошибке;
- сообщения о результате ставки.

---

### `sqlite_database.py`

Работа с локальной SQLite-базой.

Используется:

- для локальных тестов;
- как резервное хранилище;
- для отладки SQL-запросов.

---

### `postgres_database.py`

Работа с постоянной PostgreSQL-базой Neon.

После полной миграции будет основным хранилищем проекта.

Преимущества:

- одна база для Render и локального компьютера;
- данные не теряются при деплое;
- удобная аналитика;
- возможность хранить десятки тысяч сигналов.

---

### `results.py`

Проверяет результаты ранее найденных сигналов.

Логика:

```text
Общий тотал нечётный → WIN
Общий тотал чётный → LOSE
```

Модуль:

- получает сигналы со статусом `waiting`;
- ищет завершённый матч;
- считает общий тотал;
- определяет результат;
- обновляет базу;
- отправляет сообщение в Telegram.

---

### `statistics.py`

Считает аналитику.

Поддерживает:

- общую статистику;
- WIN и LOSE;
- проходимость;
- ROI;
- статистику по странам;
- статистику по лигам;
- будущую статистику по дням недели и часам.

---

### `match_classifier.py`

Определяет тип матча.

Категории:

- `men` — мужские матчи;
- `women` — женские матчи;
- `youth` — молодёжные матчи.

Молодёжные обозначения:

```text
U16
U17
U18
U19
U20
U21
U22
U23
(16)
(18)
(20)
(мол)
Youth
Junior
```

---

### `web_server.py`

Минимальный HTTP-сервер для Render.

Нужен, потому что Render Web Service требует открытый порт.

---

### `config.py`

Читает настройки из переменных окружения:

```text
BOT_TOKEN
CHAT_IDS
CHECK_INTERVAL
DATABASE_URL
```

Настоящие значения хранятся в `.env` и в Environment Variables Render.

## Источники данных

### Flashscore

Используется для получения:

- LIVE-матчей;
- четвертей;
- итогового счёта;
- статуса матча.

Используется для доставки сигналов и отчётов.

Круглосуточно запускает бота.

### Neon PostgreSQL

Постоянно хранит сигналы и статистику.

## Правила архитектуры

1. Каждый файл отвечает только за одну область.
2. Токены и пароли не хранятся в GitHub.
3. Работа с базой выполняется только через специальные функции.
4. Новая функция сначала тестируется локально.
5. После успешного теста изменения отправляются в GitHub.
6. Render автоматически разворачивает новую версию.
7. Старая рабочая логика не удаляется до проверки новой.
