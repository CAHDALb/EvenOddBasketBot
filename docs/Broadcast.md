# Административная рассылка

Раздел доступен только пользователю с тарифом `admin`.

## Сценарий

1. Администратор открывает `👑 Админ-панель → 📢 Рассылка`.
2. Выбирает публикацию или обычный Telegram-опрос.
3. Бот сохраняет исходные `chat_id` и `message_id`.
4. Предпросмотр создаётся методом `copyMessage`.
5. Администратор выбирает аудиторию и режим закрепления.
6. После финального подтверждения запускается фоновый поток рассылки.
7. Каждому активному получателю создаётся отдельная копия сообщения.
8. При выбранном закреплении вызывается `pinChatMessage`.
9. Администратор получает итоговый отчёт.

## Аудитории

- `all` — все активные ADMIN, PREMIUM и FREE;
- `premium` — только активные PREMIUM;
- `free` — только активные FREE.

Заблокированные и недоступные пользователи в рассылку не включаются.

## Файлы

- `admin_broadcast.py` — черновики, этапы и фоновая отправка;
- `telegram_bot.py` — мастер диалога;
- `telegram_keyboards.py` — кнопки мастера;
- `telegram_sender.py` — `copyMessage` и `pinChatMessage`;
- `telegram_users.py` — выбор аудитории из PostgreSQL.

## Текущие ограничения

Черновики не записываются в PostgreSQL. После перезапуска процесса незавершённый черновик удаляется. За один запуск мастер принимает одно сообщение, поэтому Telegram-альбомы пока не поддерживаются как единая публикация.
